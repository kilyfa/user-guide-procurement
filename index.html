Requestor / Staff

Kamu membuat Purchase Request (PR) untuk kebutuhan divisimu.
PR adalah permintaan pembelian — kamu tidak perlu pilih vendor.

Staff Inventory Staff GA Staff Marketing Staff IT
Yang kamu lakukan — langkah per langkah
1

Buka menu Purchase Request

Mulai dari sini untuk membuat permintaan pembelian baru

›
Menu Purchase › Purchase Requests › New
ℹ
Kamu akan melihat form kosong. Isi semua field yang bertanda Wajib sebelum submit.
Lihat PDF: How to Create a Purchase Request
2

Pilih Jenis PR (PR Type)

Tentukan jenis kebutuhan pembelianmu

›

Isi detail produk/jasa yang dibutuhkan

4

Submit PR ke PR Manager

Klik tombol Submit — PR akan masuk ke antrian approval

›
✓
Semua field sudah diisi
Periksa kembali qty, tanggal butuh, estimasi biaya
✓
Klik tombol "Submit"
Status PR berubah menjadi "To Approve"
Tunggu notifikasi dari sistem
PR Manager akan approve atau reject. Kamu akan dapat notifikasi di Odoo.
✓
Setelah approved, PR diproses oleh tim Purchasing. Kamu bisa cek status PR kapan saja di menu Purchase Requests.

Edit Draft PR (sebelum submit)

Kalau PR masih draft, kamu bisa edit kapan saja

›
Buka PR yang masih draft

Edit field yang ingin diubah
Qty, produk, tanggal butuh, dll
Simpan atau Submit
Klik Save untuk simpan draft, atau Submit untuk kirim ke approval
Lihat PDF: How to Edit a Draft PR
6

Withdraw PR (tarik kembali)

Kalau PR sudah disubmit tapi mau ditarik kembali

›
⚠
Withdraw hanya bisa dilakukan jika PR belum di-approve. Setelah approved, hubungi PR Manager untuk reject.
Lihat PDF: How to Withdraw PR
7

Jika PR di-reject

Apa yang harus dilakukan jika PR dikembalikan

›
✗
PR di-reject berarti ada yang perlu diperbaiki. Buka notifikasi untuk lihat alasan reject dari PR Manager.
Buka PR yang di-reject
Lihat komentar di chatter (bagian bawah form)
Perbaiki sesuai catatan
Edit qty, estimasi biaya, atau tambah keterangan
Submit ulang
Klik "Submit" kembali setelah revisi
✅

PR Manager

Kamu adalah level approval pertama untuk semua Purchase Request.
Tugasmu: verifikasi kebutuhan, cek budget, approve atau reject PR.

Inventory Manager GA Manager Dept. Manager
Tugas dan tanggung jawabmu
1

Terima notifikasi PR baru

Kamu akan dapat notifikasi saat ada PR yang perlu di-review

›
Menu Purchase › Purchase Requests › To Approve
ℹ
Filter "To Approve" menampilkan semua PR yang menunggu approvalmu. Klik PR untuk buka dan review.
2

Review isi PR

Verifikasi bahwa PR sudah benar dan sesuai kebutuhan

›
Produk dan qty sudah sesuai?
Pastikan barang yang diminta memang dibutuhkan
Tanggal kebutuhan realistis?
Lead time pengadaan biasanya 1–2 minggu untuk lokal
Estimasi biaya wajar?
Bandingkan dengan harga historis jika ada
Cek budget otomatis
⚠
Sistem akan menampilkan warning di chatter jika PR ini membuat total pembelian minggu ini melebihi batas weekly budget.
⟶ Warning tidak memblokir PR — kamu tetap bisa approve, tapi catat dalam pertimbanganmu.
3

Approve atau Reject

Keputusanmu menentukan apakah PR lanjut ke proses selanjutnya

›
Apa yang terjadi setelah kamu klik Approve?
📦 OPEX Local / GA / Service / Marketing: PR langsung ke Purchasing untuk dibuatkan PO → Purchasing
🌏 OPEX Import: PR ke Finance untuk approval tambahan (karena USD) → Finance
🏗️ CAPEX: PR ke Finance → lanjut ke Direksi → Finance → Direksi
🚛 Trucking: PR ke Purchasing, akan di-merge ke 1 PO per MC event → Purchasing
Jika Reject
Klik "Refuse" / "Reject"
Wajib isi alasan penolakan
Isi alasan yang jelas
Requestor akan membaca alasan ini untuk revisi
Lihat PDF: How to Approve PR

Purchasing

Kamu mengubah PR yang sudah approved menjadi Purchase Order (PO).
Tugasmu: cari vendor, bandingkan penawaran, buat PO, dan kirim ke vendor.

Staff Purchasing Purchasing Manager
Langkah-langkah membuat PO dari PR
1

Buka PR yang sudah approved

Filter PR dengan status "Approved" — siap dibuatkan PO

›
Menu Purchase › Purchase Requests › Approved
ℹ
Kamu bisa merge beberapa PR ke 1 PO jika vendor-nya sama. Pilih PR yang relevan → klik "Create Purchase Order".
2

Kirim RFQ ke minimal 3 vendor

Bandingkan penawaran sebelum konfirmasi PO

›
Buat RFQ ke vendor 1, 2, dan 3
Klik "Send by Email" untuk kirim request penawaran
Tunggu penawaran masuk
Input harga dari email vendor ke masing-masing RFQ
Bandingkan harga, lead time, syarat bayar
Odoo menampilkan tabel perbandingan otomatis
Pilih vendor terbaik → Confirm PO
PDF: How to Create RFQ PDF: How to Compare Vendors
3

Skenario 4 variasi PR → PO

Bagaimana cara handle berbagai situasi

›
SituasiYang kamu lakukanHasil
1 PR, 1 vendorSelect PR → pilih vendor → Confirm1 PO
1 PR, vendor beda per produkPisah per line → buat PO per vendor2–3 PO
Beberapa PR, vendor samaSelect semua PR → merge → Confirm1 PO gabungan
PR Trucking (dari INV+GA+MKT)Merge → isi field MC Picking (gudang tujuan)1 PO Trucking per MC event
PDF: How to create PO from RFQ PDF: How to Approve PO
4

Khusus: PR Trucking

Ada langkah tambahan untuk PR pengiriman antar gudang

›
ℹ
PR Trucking dari beberapa dept (Inventory, GA, Marketing) untuk 1 pengiriman MC → digabung jadi 1 PO ke 1 vendor trucking.
Merge semua PR Trucking terkait
Isi field "MC Picking"
Pilih MC-Out JKT yang sesuai → sistem otomatis mengisi analytic cabang tujuan
Confirm PO → kirim ke vendor trucking
⚠
PPH 23 akan dipotong otomatis oleh Finance saat pembayaran ke vendor trucking. Kamu tidak perlu hitung manual.

Inbound / Gudang

Kamu menerima barang secara fisik dan memvalidasi di sistem Odoo.
Ada 2 peran: Inbound Coordinator JKT (terima & MC-Out) dan Staff Inbound Cabang (MC-In).

Inbound Coordinator JKT Staff Inbound Cabang
Tugas Inbound Coordinator JKT
1

Terima barang dari supplier

Validasi receipt saat barang tiba di gudang JKT

›
Menu Inventory › Operations › Transfers › Receipts
Cek fisik barang sesuai PO
Qty, kondisi, dan spesifikasi barang
Klik "Validate" di receipt Odoo
Stok JKT otomatis bertambah
PDF: How to Receive Products PDF: How to Receive Import
2

Buat MC-Out ke cabang tujuan

Kirim barang dari JKT ke cabang via transit

›
Menu Inventory › Operations › Internal Transfers
JKT/Stock
→
Virtual/Intransit
→
Cabang/Stock
ℹ
Nomor dokumen MC-Out: MCOUT-JKT-YYMMXXXXX
Setelah validate → barang masuk ke Intransit, stok JKT berkurang.
⚠
Aturan penting: Barang TIDAK boleh dikirim langsung cabang ke cabang. Semua harus lewat JKT.
Lihat PDF: How to Local Receive
Tugas Staff Inbound Cabang
3

Terima MC-In di cabang

Validasi penerimaan barang dari JKT di sistem

›
Terima barang fisik dari kurir/trucking
Cek qty dan kondisi barang
Pastikan sesuai dengan dokumen MC-Out dari JKT
Buka Transfers → MCIN-CABANG-xxx → Validate
Stok cabang otomatis bertambah
✓
Setelah validate: stok cabang bertambah, Intransit berkurang. Proses selesai di sisi gudang.

Finance HQ

Kamu memproses Vendor Bill, matching ke PO, dan melakukan pembayaran ke vendor.
Juga mengurus rekonsiliasi kurs untuk pembelian USD.

Accounting Staff Accounting Manager
Langkah-langkah proses Vendor Bill
1

Terima invoice dari vendor

Input invoice vendor ke Odoo dan match ke PO

›
Menu Purchase › [PO terkait] › Create Bill
Buka PO yang barangnya sudah diterima
Klik "Create Bill"
Odoo otomatis mengisi qty dari receipt yang sudah validated
Cocokkan dengan invoice fisik dari vendor
Pastikan harga, qty, dan nomor PO sesuai
Klik "Confirm" untuk post bill
Lihat PDF: How to Upload Vendor Bill
2

Bayar vendor

Proses pembayaran sesuai payment term yang disepakati

›
Cek due date pembayaran
Lihat di tab "Payment Terms" di Vendor Bill
Klik "Register Payment"
Pilih jurnal bank yang sesuai (BCA / OCBC)
Untuk Trucking: PPH 23 dipotong otomatis
Sistem menghitung 2% (NPWP) atau 4% (non-NPWP)
3

Rekonsiliasi kurs (Import USD)

Proses selisih kurs antara tanggal PO dan tanggal pembayaran

›
ℹ
Untuk pembelian USD: kurs saat GR berbeda dari kurs saat bayar → ada realized gain/loss yang dicatat otomatis oleh sistem.
✓
Realized Gain/Loss
Otomatis dicatat saat payment matching — tidak perlu input manual
Unrealized Gain/Loss (akhir bulan)
Buka Accounting → Accounting → Adjustment Entries → isi selisih kurs akhir bulan
Lihat PDF: How to Fill Landed Cost
4
